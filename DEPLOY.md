# Kinetic Ledger — Setup & Deployment

Full-stack SaaS for freelancers: manage clients, projects, time, and invoices.

- **Web** (repo root): React 19 + Vite 6 + Tailwind v4 + React Router + TanStack Query
- **API** (`server/`): Express + TypeScript + Prisma + PostgreSQL (Supabase), JWT auth
- **Demo login:** `demo@kineticledger.app` / `demo1234`

## Run locally

```bash
# 1. Install deps (web + api)
npm install
npm install --prefix server

# 2. Configure the API
cp server/.env.example server/.env        # set DATABASE_URL (Supabase) + JWT_SECRET
npm run --prefix server prisma:generate
npm run --prefix server prisma:migrate     # apply migrations (creates a new one if the schema changed)
npm run --prefix server seed               # demo account + sample data

# 3. Run both servers together
npm run dev:all                            # web :3000  +  api :4000
```

`VITE_API_URL` (web) defaults to `http://localhost:4000`; override via a root `.env` for production.

## Continuous integration

`.github/workflows/ci.yml` runs on every push / PR to `main`: typecheck + build for both the web app and the API.

## Deployment

### Database — Supabase
Use the project's connection string. The **Session pooler** URL works for both migrations and the app over IPv4.

### API — Render
1. New **Blueprint** → point at this repo (uses `render.yaml`, root `server/`).
2. Set env vars in the dashboard:
   - `DATABASE_URL` — Supabase connection string
   - `FRONTEND_URL` — your Vercel URL (for CORS), e.g. `https://kinetic-ledger.vercel.app`
   - `JWT_SECRET` — auto-generated by the blueprint
3. The build runs `prisma migrate deploy` automatically (applies any pending migrations) before `prisma generate` + build, so schema changes ship on every push. Seed demo data once with `npm run seed` if desired.

> Migrations: the existing DB was baselined as `0_init`. For future schema changes, edit `schema.prisma`, run `npm run --prefix server prisma:migrate` locally to create a migration, and push — Render applies it on deploy.

### Web — Vercel
1. Import the repo. Vercel auto-detects Vite (`vercel.json` sets the SPA rewrite for React Router).
2. Set env var `VITE_API_URL` = your Render API URL, e.g. `https://kinetic-ledger-api.onrender.com`.
3. Deploy. Update the API's `FRONTEND_URL` to the final Vercel domain so CORS matches.

> Note: Render's free tier sleeps after inactivity — the first request after idle takes a few seconds to wake.
