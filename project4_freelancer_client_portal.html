Project 4 — Full-Stack SaaS

Freelancer Client Portal

A full-stack SaaS dashboard where freelancers manage clients, track projects, log time, and generate professional PDF invoices — all in one place.

React + Node.js PostgreSQL + Prisma PDF Invoice Gen High Upwork Demand

🎯 What It Does

Freelancers spend time chasing invoices, updating spreadsheets, and manually calculating hours. This portal eliminates all of that. After signing up, a freelancer gets a private dashboard where they can add clients, create projects under each client, log time per project, and then generate a professional PDF invoice in one click — pre-filled with all the hours and rates already in the system.

👥
Client Management
Add, edit, and organise all clients in one place. Each client has their own profile, project history, and invoice archive.
📁
Project Tracking
Create projects under any client with hourly rate, status (active / paused / completed), and linked time logs.
⏱️
Time Logging
Log time entries per project — description, hours worked, date. Running totals update automatically on the dashboard.
🧾
PDF Invoice Generator
Select a client, add line items, click "Generate PDF". Professional invoice downloads instantly using pdf-lib — no backend required for rendering.
📊
Earnings Dashboard
See total earnings, this month's revenue, outstanding invoice amounts, and active project count at a glance. Bar chart by month using Recharts.
🔐
Secure Auth
JWT-based authentication with bcrypt password hashing. Every piece of data is scoped to the logged-in user — no data leaks between accounts.

🏗️ Architecture

Three-layer architecture: a React SPA on Vercel, an Express REST API on Render, and a Supabase PostgreSQL database accessed through Prisma ORM.

💡
PDF generation happens entirely in the browser using pdf-lib. The backend never needs to render or store PDF files — it just returns the invoice data as JSON, and the frontend renders the PDF and triggers a download. This keeps your Render plan free-tier friendly.

🗄️ Database Schema (Prisma)

Six models covering the full domain. Every model except InvoiceItem has a createdAt timestamp. All user data is isolated by userId — never expose cross-user data.

// schema.prisma

generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}

model User {
  id           String    @id @default(cuid())
  email        String    @unique
  passwordHash String
  name         String
  businessName String?
  logoUrl      String?
  currency     String    @default("USD")
  createdAt    DateTime  @default(now())
  clients      Client[]
  invoices     Invoice[]
}

model Client {
  id        String    @id @default(cuid())
  userId    String
  user      User      @relation(fields: [userId], references: [id])
  name      String
  email     String?
  company   String?
  phone     String?
  createdAt DateTime  @default(now())
  projects  Project[]
  invoices  Invoice[]
}

model Project {
  id          String      @id @default(cuid())
  clientId    String
  client      Client      @relation(fields: [clientId], references: [id])
  title       String
  description String?
  status      ProjectStatus @default(ACTIVE)
  hourlyRate  Float       @default(0)
  totalBudget Float?
  createdAt   DateTime    @default(now())
  timeEntries TimeEntry[]
}

enum ProjectStatus {
  ACTIVE
  PAUSED
  COMPLETED
}

model TimeEntry {
  id          String   @id @default(cuid())
  projectId   String
  project     Project  @relation(fields: [projectId], references: [id])
  description String
  hours       Float
  date        DateTime
  createdAt   DateTime @default(now())
}

model Invoice {
  id            String        @id @default(cuid())
  userId        String
  user          User          @relation(fields: [userId], references: [id])
  clientId      String
  client        Client        @relation(fields: [clientId], references: [id])
  invoiceNumber String        @unique
  status        InvoiceStatus @default(DRAFT)
  totalAmount   Float
  dueDate       DateTime
  issuedAt      DateTime      @default(now())
  paidAt        DateTime?
  notes         String?
  items         InvoiceItem[]
}

enum InvoiceStatus {
  DRAFT
  SENT
  PAID
  OVERDUE
}

model InvoiceItem {
  id          String  @id @default(cuid())
  invoiceId   String
  invoice     Invoice @relation(fields: [invoiceId], references: [id])
  description String
  quantity    Float
  unitPrice   Float
  total       Float   // quantity * unitPrice, stored for easy querying
}
⚠️
When querying invoices for the dashboard stats, always filter by userId via a join through User. Never trust a raw clientId from the request body — always verify that the client belongs to the authenticated user before associating data.

🔌 API Endpoints

All routes under /api except /api/auth require a valid Authorization: Bearer <token> header. The auth middleware attaches req.userId to every request.

🔐 Auth
POST
/api/auth/register
Create account. Body: { name, email, password }. Returns JWT token.
POST
/api/auth/login
Login. Body: { email, password }. Returns JWT token + user object.
GET
/api/auth/me
Returns currently authenticated user's profile. Requires auth.
📊 Dashboard
GET
/api/dashboard/stats
Returns: totalEarnings, thisMonthEarnings, outstandingAmount, activeProjects, totalClients, monthlyData[] (12 months for chart).
👥 Clients
GET
/api/clients
List all clients for the logged-in user. Optional ?search= query param.
POST
/api/clients
Create a client. Body: { name, email?, company?, phone? }.
GET
/api/clients/:id
Get a single client + all their projects + recent invoices.
PUT
/api/clients/:id
Update client details.
DELETE
/api/clients/:id
Delete client and cascade-delete all their projects, time entries, invoices.
📁 Projects
GET
/api/clients/:clientId/projects
List all projects under a client. Returns project + aggregated hours.
POST
/api/clients/:clientId/projects
Create project. Body: { title, description?, hourlyRate, totalBudget? }.
GET
/api/projects/:id
Get project detail + time entries + total hours billed.
PUT
/api/projects/:id
Update project. Body: { title?, description?, hourlyRate?, status?, totalBudget? }.
DELETE
/api/projects/:id
Delete project and cascade-delete all time entries.
⏱️ Time Entries
GET
/api/projects/:projectId/time-entries
List all time entries for a project, ordered by date desc.
POST
/api/projects/:projectId/time-entries
Log time. Body: { description, hours, date }.
PUT
/api/time-entries/:id
Update time entry.
DELETE
/api/time-entries/:id
Delete time entry.
🧾 Invoices
GET
/api/invoices
List all invoices for user. Optional ?status= and ?clientId= filters.
POST
/api/invoices
Create invoice. Body: { clientId, dueDate, notes?, items: [{ description, quantity, unitPrice }] }. Auto-assigns invoiceNumber and computes totalAmount.
GET
/api/invoices/:id
Get invoice detail with all items and client info (for PDF generation).
PUT
/api/invoices/:id
Update invoice status or details. Body: { status?, dueDate?, notes?, items? }.
PATCH
/api/invoices/:id/mark-paid
Set invoice status to PAID and record paidAt timestamp.
DELETE
/api/invoices/:id
Delete invoice and all its items. Only allow if status is DRAFT.
⚙️ Settings
GET
/api/settings
Return user profile (name, businessName, currency, logoUrl).
PUT
/api/settings
Update user settings. Body: { name?, businessName?, currency?, logoUrl? }.

🔨 Build Order (10 Steps)

  • 1
    Backend scaffold + auth endpoints
    Init Express + TypeScript project. Set up Prisma, connect to Supabase. Implement POST /auth/register and POST /auth/login with bcrypt + JWT. Write the auth middleware that reads the Bearer token and attaches userId to req. Test with Postman before touching the frontend.
  • 2
    Frontend scaffold + auth pages
    Init React + TS + Vite. Install Tailwind, shadcn/ui, React Router, TanStack Query, Axios. Build Login and Register pages. Set up Axios interceptor that attaches Authorization header from localStorage. Implement protected route wrapper component.
  • 3
    Client CRUD (backend + frontend)
    Build all 5 client endpoints. On the frontend: Clients list page with search, Add/Edit client dialog (shadcn Dialog + React Hook Form), delete confirmation dialog. Use TanStack Query useQuery + useMutation for all API calls.
  • 4
    Project CRUD (backend + frontend)
    Build project endpoints. Client detail page: shows client info + projects list + empty invoice section. Add/Edit project form. Project status badge (Active = green, Paused = amber, Completed = grey).
  • 5
    Time logging (backend + frontend)
    Build time entry endpoints. Project detail page: time entries table with add/edit/delete. Show total hours and calculated earnings (hours × hourly rate) in project header. Use date-fns for date formatting.
  • 6
    Dashboard stats + Recharts
    Build /api/dashboard/stats. Calculate: totalEarnings (sum of PAID invoices), thisMonthEarnings, outstandingAmount (SENT invoices), activeProjects count. Build 12-month array for bar chart. On frontend: 4 stat cards + BarChart from Recharts.
  • 7
    Invoice backend (create + manage)
    Build all invoice endpoints. Auto-increment invoice number: query max existing number and increment. Validate that clientId belongs to the user. InvoiceItem totals calculated server-side. Invoice list with status filter tabs.
  • 8
    PDF invoice generator (client-side pdf-lib)
    Fetch GET /api/invoices/:id — returns full invoice + client + items. Use pdf-lib to build the PDF: header with business name, client info block, itemised table, total row, due date + notes. Trigger download via URL.createObjectURL. Test across Chrome, Firefox, Safari.
  • 9
    Settings page + logo on PDF
    Settings page with name, business name, currency selector. Logo upload — store as base64 string in User.logoUrl (keep small: <50KB). Pull logo into pdf-lib when generating invoices. Update the PDF template to include logo in top-left corner.
  • 10
    Deploy + GitHub Actions CI
    Deploy backend to Render (set DATABASE_URL, JWT_SECRET env vars). Deploy frontend to Vercel (set VITE_API_URL). Add CORS origin whitelist on backend. Set up GitHub Actions: lint + tsc --noEmit on push to main. Test the full flow end-to-end in production.

CONSTRAINTS & NOTES
===================
- Database is Supabase PostgreSQL. Use the connection pooler URL (port 6543) for Prisma in production to avoid connection limits on the free tier. Direct URL (port 5432) for migrations only.
- PDF generation is entirely client-side (pdf-lib in the browser). Never send invoice data to a PDF rendering service.
- JWT expiry: 7 days. No refresh token for MVP — force re-login after expiry.
- All monetary values stored as Float in PostgreSQL (this is fine for a portfolio project — in production you'd use Decimal or store in cents).
- Invoice number format: INV-001, INV-002 etc. On create: query MAX(invoiceNumber) for the user, extract the number, increment by 1, zero-pad to 3 digits.
- Cascade deletes: deleting a client should delete all their projects, time entries, invoices, and invoice items. Set up Prisma onDelete: Cascade on all foreign keys.
- CORS: backend must whitelist the Vercel frontend URL. Use an env var FRONTEND_URL for this.
- Logo stored as base64 in the database (acceptable for portfolio, max 50KB enforced on upload).
